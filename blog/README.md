# Blog folder

Drop a markdown (`.md`) file in here for each post. The Blog page builds a tree
organized by **publish day**, newest first. Clicking a post opens it rendered in
a new tab.

## Frontmatter

```markdown
---
title: Hello, world
published: 2026-06-27 09:00
description: The first entry — what this is and where it's going.
---

# Hello, world

...your post in normal markdown...
```

| Field         | Required | Purpose                                             |
| ------------- | -------- | --------------------------------------------------- |
| `title`       | yes      | Post title (the tree leaf).                         |
| `published`   | yes      | Timestamp `YYYY-MM-DD HH:MM` — sets the day & time. |
| `description` | no       | Short excerpt (shown on hover).                     |

`published` drives both the grouping (by day) and the order (newest first). The
date portion becomes a day heading; the time shows next to the post.

The manifest (`blog/index.json`) is regenerated by the `Build indexes` GitHub
Action on push, or locally with `node scripts/build-index.mjs`.
